## 品牌档案信息架构

每个品牌的 `.md` 文件应包含以下 **5个模块**。前3个是品牌的**独立商业画像**，后2个是**竞品分析视角**。

```
品牌档案
├── ① 品牌身份          ← 谁是这家公司、做什么产品（独立知识）
├── ② 产品与技术体系    ← 产品矩阵、技术路线、核心参数（独立知识）
├── ③ 市场与商业        ← 市场地位、用户画像、商业模式、财务（独立知识）
├── ④ 竞品评估          ← 与同类品牌的对比、优劣势、壁垒（竞品视角）
└── ⑤ 与我们的关系      ← 对标关系、学习/警惕、对我们产品的启示
```

### ① 品牌身份（必填）

> 目标：不看外部资料就能快速理解这家公司的基本面。

| 字段 | 说明 | 必要性 |
|------|------|--------|
| 一句话定位 | 做什么、凭什么赢 | 必填 |
| 公司全称/英文名 | — | 必填 |
| 成立时间 | — | 必填 |
| 总部 | — | 必填 |
| 创始人/核心团队 | 关键人物及背景 | 选填 |
| 员工规模 | — | 选填 |
| 上市/融资 | 代码/轮次/金额/估值 | 选填 |
| 投资方 | 谁投了他们（链接 [[07_investors/]]） | 选填 |
| 官网 | URL | 必填 |

### ② 产品与技术体系（必填）

> 目标：完整理解这个品牌的产品布局和技术能力边界。

| 内容 | 说明 | 必要性 |
|------|------|--------|
| 产品矩阵 | 全部在售产品（型号、定位、核心参数、价格、状态） | 必填 |
| 技术路线 | 用什么技术？自研还是外采？核心差异化在哪 | 必填 |
| 核心参数展开 | 旗舰产品的详细参数表 | 选填 |
| 技术原理 | 关键技术的工作原理（如有价值） | 选填 |
| 产品路线图 | 已知/传闻的下一代产品 | 选填 |
| App/软件生态 | 配套App功能、平台、订阅服务 | 选填 |

### ③ 市场与商业（建议填）

> 目标：理解这个品牌的商业逻辑——卖给谁、怎么卖、赚多少钱。

| 内容 | 说明 | 必要性 |
|------|------|--------|
| 市场地位 | 份额/排名/增长 | 建议填 |
| 目标用户 | 卖给谁？用户画像 | 建议填 |
| 商业模式 | 硬件一次性/订阅/B2B/B2B2C | 建议填 |
| 定价策略 | 价格区间、历史价格变化 | 选填 |
| 销售渠道 | DTC/零售/电商/医疗渠道 | 选填 |
| 财务数据 | 营收/利润/增长率 | 选填 |
| 战略合作 | 关键合作伙伴（供应商/分销商/医院） | 选填 |

### ④ 竞品评估（建议填）

> 目标：在同品类中定位这个品牌的竞争力。

| 内容 | 说明 | 必要性 |
|------|------|--------|
| 核心优势 | 2-3个真正的差异化优势 | 建议填 |
| 劣势/风险 | 弱点、潜在风险 | 建议填 |
| 技术壁垒 | 专利/独有技术/认证 | 选填 |
| 监管认证 | FDA/CE/NMPA | 选填 |
| 最新动态 | 近6个月重要事件 | 选填 |

### ⑤ 与我们的关系（必填）

> 目标：明确这个品牌对我们项目的意义。

| 字段 | 说明 |
|------|------|
| 对标关系 | 直接竞品 / 间接竞品 / 技术参考 / 合作对象 / 供应链相关 |
| 关注优先级 | 高/中/低 |
| 核心关注点 | 最该学什么/警惕什么 |
| 对我们产品的启示 | 具体的产品/功能/策略借鉴 |

## 与供应商档案的区别

| 维度 | 品牌档案（06_brands） | 供应商档案（05_vendors） |
|------|----------------------|------------------------|
| **关注核心** | 市场定位、用户价值、商业逻辑 | 产品能力、技术参数、商务条件 |
| **产品视角** | 卖给谁、怎么定价、怎么营销 | 能不能用在我们的产品里、参数达标吗 |
| **技术深度** | 概览级（知道用什么技术即可） | 工程级（需要规格书、参考设计） |
| **商业部分** | 商业模式、营收、市场份额 | 报价、交期、MOQ |
| **独特模块** | 用户画像、营销策略、监管认证 | Demo测试、拜访记录、商务合同 |

## 使用指南

### 新增品牌
1. 确定所属品类目录（Ring/CGM/Watch/BCI/Earbuds/Glasses）
2. 复制 [[.track_template/brand_template/|模板目录]] 为 `<品牌名>/`
3. 重命名 `brand.md` 为 `<品牌名>.md`，按5模块架构填写
4. 至少完成 **①②⑤** 三个必填模块
5. 更新品类 `brand_matrix.md` 和本 README

### 补充现有品牌
1. 对照完整性评估表，找到标记为 ⚠️ 或 — 的模块
2. 按优先级补充：③市场商业 > ④竞品评估 > ②产品技术（展开）
3. 补充后更新完整性评估表

### 品牌档案 vs wiki 的分工

| 需求 | 去哪 |
|------|------|
| 某品牌的完整画像 | `06_brands/<品类>/<品牌>/<品牌>.md` |
| 跨品牌对比 | `wiki/topics/` |
| 某技术概念 | `wiki/concepts/` |
| 品牌实体简要介绍 | `wiki/entities/<品牌>.md` |

**规则**：单品牌纵向 → `06_brands/`；跨品牌横向 → `wiki/topics/`。
