# Brand 模板

## 设计理念

**一个品牌 = 一个目录，索引页 + files/。**

新增品牌时，把整个 `template/files/` 目录复制过去即可。

```
📂 4_brands/
    📂 <品牌名>/
        - <品牌名>.md          ← 品牌档案（结构化认知 + 资料入口）
        📂 files/              ← 原始资料（按下方分类存放）
```

### 分类原则

- **不确定放哪 → 放 9_新闻**，后续再归类
- **不确定有没有用 → 先留着放 0_归档**，定期清理
- **一个文件涉及多个分类 → 放最相关的那个**，在索引页交叉引用
- **子目录用不用随缘**，没素材就空着，不需要每个都填

## 使用规范

### 品牌档案只放什么

| 放 | 不放 |
|----|------|
| 结构化数据（表格、参数） | PDF/文章全文复制 |
| 你的判断和结论 | 未消化的原始信息 |
| 资料文件链接 | 品牌间横向对比（那是 wiki 的事） |
| 近期动态摘要 | 过期新闻（超过 6 个月归档） |

### 品牌目录 vs wiki 的分工

| 需求 | 去哪 |
|------|------|
| 某品牌的完整画像 | `4_brands/<品牌>/<品牌>.md` |
| 跨品牌对比（如 Abbott vs Dexcom） | `wiki/topics/` |
| 某技术概念的深度解释 | `wiki/concepts/` |
| 品牌作为实体的简要介绍 | `wiki/entities/<品牌>.md` |

> **简单规则**：单品牌纵向 → `4_brands/`；跨品牌横向 → `wiki/topics/`。

### 文件命名

- 资料文件：`类型-简述.扩展名`（如 `规格书-FreeStyle-Libre-3.pdf`）
- 保持原始文件名也可，但索引页中的链接必须带一句话说明

## 新增品牌流程

1. 复制整个 `template/` 目录，重命名为 `<品牌名>/`
2. 将 `brand.md` 重命名为 `<品牌名>.md`，填入基础信息
3. 收集到的资料放入 `files/` 对应子目录
4. 在索引页中引用资料并注明价值
5. 如有跨品牌对比需求，在 `wiki/topics/` 创建对比页
6. 定期清理：空目录不影响使用，`0_归档/` 每季度清理一次
