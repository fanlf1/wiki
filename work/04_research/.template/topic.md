# 课题研究模板

> **用途**：每个研究子课题一份。课题从 02_demand 的需求推导而来，目标是用论文和实验数据回答"这个功能技术上做得出来吗"。

## 目录结构

```
04_research/
├── <研究方向>/              # 第一层：BCI, CGM, HRV-平静度 等
│   ├── overview.md          # 该方向的总入口（指向下属 topics）
│   └── <topic名称>/         # 第二层：具体研究课题
│       ├── topic.md         # 课题总结（本文件）
│       ├── 1_news/          # 原始素材（clippings、新闻、行业报告）
│       ├── 2_analysis/      # 分析笔记（信息校准、交叉验证、去重）
│       ├── 3_paper/         # 论文（PDF、DOI链接、论文笔记）
│       └── 4_output/        # 最终产物（方案建议、选型结论、决策建议）
```

```markdown

demand: "[[../../02_demand/对应场景/reports/xxx报告]]"
demand-reqs: [REQ-xxx]
tags: [技术标签]
---

| 维度 | 评估 | 说明 |
|------|------|------|

| 硬件约束 | 无/有/严重 | 功耗、体积、成本是否允许 |
```

## 课题生命周期

```
发现 → 调研中 → 已验证 / 不可行 / 暂缓
  │        │
  │        └── 素材充足，启动深度分析
  └── inbox clipping 匹配，创建 topic，status=发现
```

| 状态 | 含义 | 触发条件 |
|------|------|---------|
| **发现** | 初步识别，素材收集中 | inbox clipping 匹配时自动创建 |
| **调研中** | 深度研究进行中 | 素材 ≥3 条 或 用户确认启动 |
| **已验证** | 结论明确，技术可行 | 可行性评估完成 |
| **不可行** | 结论明确，技术上做不出 | 论文/实验否决 |
| **暂缓** | 当前优先级不足 | 用户决策 |

## 子目录用途

| 目录 | 放什么 | 不放什么 |
|------|--------|---------|
| `1_news/` | 原始剪报、行业新闻、厂商公告、未经分析的材料 | 分析笔记、最终结论 |
| `2_analysis/` | 信息校准笔记、交叉验证、信源评级、对比分析 | 原始剪报、论文原文 |
| `3_paper/` | 论文PDF、DOI链接、论文笔记（`论文名-笔记.md`） | 新闻、非学术资料 |
| `4_output/` | 方案建议、选型结论、给 02_demand 的决策建议 | 过程文件、原始素材 |

## 使用指南

- **每个 topic 对应一个具体的研究子问题**，而非整个研究方向
- **课题与需求的关系必须显式标注**：demand-reqs 字段链接到具体 REQ
- **topic.md 是活文档**：随研究进展持续更新，不是一次性写完的
- **证据链要可追溯**：每个结论都应链接到 1_news/2_analysis/3_paper 中的来源
