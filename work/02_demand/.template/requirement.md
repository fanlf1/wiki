# 产品需求模板

> **用途**：从研究报告中推导出的可执行产品需求。
> 命名：`REQ-<编号>-<简述>.md`（如 `REQ-001-血糖戒指续航不低于5天.md`）

```markdown
---
type: requirement
id: REQ-001
created: YYYY-MM-DD
updated: YYYY-MM-DD
status: 草稿 | 已确认 | 已排期 | 已实现
priority: P0 | P1 | P2
source: [[reports/xxx报告]]
tags: [相关标签]
---
```

## 需求优先级定义

| 优先级 | 含义 | 示例 |
|--------|------|------|
| P0 | 没有这个产品不成立 | 血糖测量MARD < 15% |
| P1 | 核心体验必须有 | 续航 ≥ 5天 |
| P2 | 加分项，可以迭代 | 皮肤温度趋势分析 |

## 从 report 到 requirement 的路径

1. 研究报告中"对我们产品的建议" → 提取为 requirement
2. 每个 requirement 必须能追溯到具体的 report 和 card
3. 验收标准必须可量化、可测试
