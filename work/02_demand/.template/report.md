# 研究报告模板

> **用途**：定期汇总洞察卡片，形成有结论的研究报告。
> 命名：`YYYY-MM-DD-研究主题.md`（如 `2026-05-26-无创血糖戒指用户付费意愿研究.md`）

```markdown
---
type: report
created: YYYY-MM-DD
updated: YYYY-MM-DD
status: 草稿 | 完成 | 已转化为需求
cards: [引用的洞察卡片数量]
tags: [相关标签]
---

# <研究主题>

## 研究问题
> 用一句话描述这份报告要回答的核心问题。

---

## 核心结论（3条以内）
1. ...
2. ...
3. ...

- 为什么要做这个研究
- 研究范围和时间跨度

## 发现

- 数据支撑

- 解读

## 引用的洞察卡片
- [[cards/2026-05-26-xxx]]
- [[cards/2026-05-26-xxx]]
```

## 使用指南

- **从卡片聚合**：报告中每个"发现"都应该能追溯到具体的洞察卡片
- **结论前置**：核心结论放在最前面，让读者5秒内知道答案
- **可执行性**：最后的"建议"必须能转化为 requirements/
- **标注未回答问题**：比假装全知道更有价值
