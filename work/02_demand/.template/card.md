# 洞察卡片模板

> **用途**：快速记录一条洞察。一个洞察一个文件，不需要完美，记下来就行。
> 命名：`YYYY-MM-DD-关键词.md`（如 `2026-05-26-用户不愿为血糖戒指付月费.md`）

```markdown
---
type: card
created: YYYY-MM-DD
scenario: blood-sugar/daily-monitoring（分类/子场景目录名）
persona: 对哪类用户有意义（如 T2D患者 / 健身人群 / 高净值人群）
source: 观察来源（访谈/问卷/新闻/数据/竞品分析/直觉）
confidence: 高 | 中 | 低
tags: [相关标签]
---

# <一句话标题>

## 洞察内容
> 用2-3句话描述你观察到的事实或现象。

## 证据
- 证据1
- 证据2

## 意味着什么
- 对我们产品的启示（1-2句）

## 相关卡片
- [[../其他场景/cards/xxx]]（跨场景关联时用相对路径）
```

## 使用指南

| 字段 | 说明 |
|------|------|
| scenario | 这条洞察属于哪个场景，和目录名对应 |
| persona | 哪类用户的洞察，引用 [[../_shared/personas/xxx]] |
| source | 这条洞察从哪来的，方便回溯 |
| confidence | 你对这条洞察的把握程度，低置信度也值得记 |
| 意味着什么 | **最重要的一栏**——光记录事实不够，必须写出"so what" |

### 好的卡片 vs 差的卡片

**好的**：`2026-05-26-T2D患者希望血糖趋势比绝对值更有用.md`
- 场景明确（blood-sugar-daily）
- 人物明确（T2D患者）
- 有具体现象 + so what

**差的**：`2026-05-26-市场洞察.md`
- 没有场景归属
- 没有目标用户
- 塞了5个不相关的观察
