# Wiki 项目管理区

> 本目录用于管理各类项目，是小清和初心协同工作的项目中心。

---

## 📁 目录结构

```
wiki/projects/
├── README.md              # 本文件 - 项目管理说明
├── CLAUDE.md              # 项目管理规则（小清的工作指南）
├── active/                # 活跃项目（正在进行的项目）
├── archived/              # 已归档项目（已完成或暂停的项目）
└── templates/             # 项目模板（标准项目结构）
```

---

## 🎯 项目管理原则

### 1. 项目分类

**按状态分类**：
- **active/**：正在进行的活跃项目
- **archived/**：已完成或暂停的归档项目

**按类型分类**（建议在项目命名中体现）：
- **research**：研究调研类项目
- **development**：技术开发类项目
- **evaluation**：评估分析类项目
- **experiment**：实验验证类项目

### 2. 标准项目结构

每个项目应包含以下文件：

```
project-name/
├── README.md              # 项目概述
├── CLAUDE.md              # 项目特定规则（可选）
├── tasks.md               # 任务清单
├── decisions.md           # 技术决策记录（可选）
├── meetings.md            # 会议记录（可选）
└── resources/             # 项目资源文件
```

### 3. 项目命名规范

- 格式：`YYYY-MM-DD-项目名称`
- 示例：`2026-04-28-CGM血糖监测POC验证`

---

## 📋 项目生命周期

```
创建（templates/）→ 活跃（active/）→ 归档（archived/）
```

### 创建阶段
1. 从templates/复制标准项目结构
2. 在active/创建项目目录
3. 编写README.md定义项目目标

### 活跃阶段
1. 更新tasks.md任务清单
2. 记录decisions.md技术决策
3. 定期更新项目状态

### 归档阶段
1. 项目完成后移至archived/
2. 编写项目总结
3. 清理临时文件

## 🚀 快速开始

### 创建新项目

```bash

cd /mnt/c/code/wiki/projects/active

# 2. 创建项目目录（使用日期前缀）
mkdir 2026-04-28-我的项目

# 3. 复制模板
cp ../templates/project-template.md ./2026-04-28-我的项目/README.md

# 4. 编辑项目概述
# （编辑README.md）
```

### 查看活跃项目

```bash
ls -la /mnt/c/code/wiki/projects/active/
```

### 归档项目

```bash
# 移动项目到归档区
mv /mnt/c/code/wiki/projects/active/2026-04-28-我的项目 \
   /mnt/c/code/wiki/projects/archived/
```
