**我们的疑问**：
> 睿芯微CIS使用什么接口输出高光谱数据？

**期望回答**：
- [ ] MIPI CSI-2（标准接口）
- [ ] 并行接口（DVP/Parallel）
- [ ] SPI接口
- [ ] USB接口
- [ ] 其他自定义接口

**关键参数**（如果是MIPI CSI-2）：
- [ ] 支持的lane数：1-lane? 2-lane? 4-lane?
- [ ] 数据速率：多少Mbps?
- [ ] 协议版本：CSI-2 v1.0? v2.0?
- [ ] 数据格式：RAW? YUV? 自定义?

#### 问题2：BES2800BP平台兼容性 ⚠️

**我们的发现**：
> 经过查阅BES2800BP官方datasheet（Rev 0.41，48页），**未发现CSI接口相关说明**，仅提及MIPI DSI显示接口。

**需要睿芯微确认**：
- [ ] 睿芯微CIS**必须**通过CSI接口连接吗？
- [ ] 是否有**非CSI接口**的连接方案？
- [ ] 睿芯微是否**已验证过**BES2800平台？
  - 如果是，是BES2800的哪个具体型号？（BP？其他？）
  - 使用的什么接口连接？
  - 是否有参考设计？

**可能的连接方案**（请睿芯微评估可行性）：

**方案A：通过MIPI CSI-2连接**（如果BES2800BP有未公开的CSI）
```
摄像头 ──MIPI CSI-2──> 睿芯微CIS ──MIPI CSI-2──> BES2800BP (?)
                                           (需要确认)
```

**方案B：通过并行接口连接**
```
摄像头 ──MIPI CSI-2──> 睿芯微CIS ──并行DVP──> BES2800BP (?)
                                           (需要确认BES2800BP是否支持)
```

**方案C：通过SPI接口连接**
```
摄像头 ──MIPI CSI-2──> 睿芯微CIS ──SPI──> BES2800BP
                                           (需要确认速率是否足够)
```

**方案D：通过USB接口连接**
```
摄像头 ──MIPI CSI-2──> 睿芯微CIS ──USB──> BES2800BP
                                           (CIS作为USB设备)
```

## ⚠️ 关键风险提醒

### 技术风险

**风险1：BES2800BP接口不兼容**
- 概率：中-高
- 影响：需要切换到ESP32-P4或高通平台，延迟1-2个月
- 缓解措施：
  - 会议中必须确认BES2800BP连接方案
  - 并行启动ESP32-P4技术准备
  - 评估切换平台的成本和时间

**风险2：评估板交付延迟**
- 概率：中
- 影响：POC验证延迟，影响Prototype时间表
- 缓解措施：
  - 会议中确认交付周期
  - 争取优先交付
  - 如果延迟，考虑先获取技术资料进行仿真

### 商务风险

**风险3：MOQ过高或价格超预算**
- 概率：低-中
- 影响：成本超支，需要重新评估方案
- 缓解措施：
  - 会议中确认样品和小批量价格
  - 争取POC阶段的小批量支持
