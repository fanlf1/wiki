# Vendor 模板

## 设计理念

**一个供应商 = 一个目录，索引页 + files/。**

供应商和品牌的关注点不同：品牌侧重"市场理解"，供应商侧重"产品评估 + 商务往来"。

```
📂 5_vendors/
    📂 <Vendor名称>/
        - <Vendor名称>.md       ← 供应商档案（评估结论 + 资料入口）
        📂 files/               ← 原始资料（按下方分类存放）
```

## files/ 目录结构

```
📂 files/
├── 📂 1_公司/
│   ├── 背景/              公司简介、官网资料、年报
│   ├── 团队/              核心团队、关键联系人、组织架构
│   ├── 资质/              ISO认证、质量体系、资质证书
│   └── 战略/              战略方向、投资者关系、行业地位
│
├── 📂 2_产品/
│   ├── 规格书/            Datasheet、产品规格、参数表
│   ├── 参考设计/          Reference Design、Schematic、BOM、Gerber
│   ├── 手册/              用户手册、Application Note、驱动文档
│   ├── 选型指南/          产品线对比、选型矩阵
│   └── 路线图/            产品路线图、未来规划
│
├── 📂 3_评估/
│   ├── Demo/              Demo板照片、评估套件、测试代码
│   ├── 测试报告/          测试数据、Benchmark、性能对比
│   └── 技术分析/          技术深度分析、可行性评估、专利分析
│
├── 📂 4_商务/
│   ├── 报价/              报价单、价格表、阶梯价格
│   ├── 合同/              NDA、采购合同、框架协议
│   ├── 通讯记录/          邮件往来、会议纪要、电话记录
│   └── 名片/              名片扫描、联系方式、对接人信息
│
├── 📂 5_拜访/             现场拜访记录、工厂参观照片、交流纪要
│
├── 📂 6_新闻/             供应商相关新闻、动态
│
└── 📂 0_归档/             过期资料、被取代的旧版本
```

### 分类原则

- **不确定放哪 → 放 6_新闻**，后续再归类
- **不确定有没有用 → 先放 0_归档**，定期清理
- **技术评估过程产生的所有文件 → 放 3_评估/**
- **商务相关的一切 → 放 4_商务/**
- **供应商 vs 品牌的区别**：供应商更关注产品能不能用、怎么用、多少钱；品牌更关注市场定位和竞争格局

### 与品牌模板的差异

| 维度 | 品牌模板 | 供应商模板 |
|------|----------|-----------|
| 核心关注 | 市场定位、竞争格局 | 产品能力、商务条件 |
| 技术部分 | 技术壁垒（概览级） | 规格书、参考设计、Demo（工程级） |
| 独有分类 | 营销、评测、监管 | 报价、合同、名片、拜访 |
| 索引页重点 | 竞争判断 | 合作评估 |

## 使用规范

### 索引页只放什么

| 放 | 不放 |
|----|------|
| 评估结论和判断 | Datasheet全文（放files/） |
| 关键参数摘要 | 完整报价单（放files/） |
| 交流结论 | 会议原始纪要（放files/） |
| 对比结论 | 每次通讯的详细内容 |

### 新增供应商流程

1. 复制整个 `template/` 目录，重命名为 `<Vendor名称>/`
2. 将 `vendor.md` 重命名为 `<Vendor名称>.md`，填入基础信息
3. 收集到的资料放入 `files/` 对应子目录
4. 在索引页中引用资料并注明价值
