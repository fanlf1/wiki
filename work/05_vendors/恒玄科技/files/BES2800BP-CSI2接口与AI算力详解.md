# BES2800BP — 摄像头接口与 AI 算力详解

> 聚焦 BES2800BP 的摄像头接口能力（CSI-2 疑问与替代方案）与 AI/NPU 运算能力，与 STM32N675、高通 Snapdragon Wear 平台进行系统对比，为无创血糖戒指/手表的 SoC 选型提供决策依据。
>
> **⚠️ 重要更正**：经核实 [官方 Brief Datasheet](https://www.bestechnic.com/Uploads/keditor/file/20241010/20241010192510_70770.pdf)，BES2800BP 的外设列表中**未包含 MIPI CSI / 摄像头接口**，仅有 MIPI DSI（显示）和 SCI（串行通信）。此前关于 CSI-2 的信息来自投资者口头回复，非官方文档。本报告已做相应修正。

## 1. BES2800BP 核心架构

### 1.1 芯片概览

| 项目 | 规格 |
|------|------|
| 制程 | 6nm FinFET |
| 定位 | 超低功耗智能可穿戴 SoC |
| 无线连接 | 双模蓝牙 5.4 + Wi-Fi 6（可选） |
| 封装 | SIP（集成 Flash / PSRAM） |
| 应用 | 智能手表、智能眼镜、AI 耳机 |
| 官方页面 | [Bestechnic BES2800BP](https://www.bestechnic.com/en/article/72/53.html) |

### 1.2 处理器子系统

BES2800BP 采用**双子系统架构**，每组各配独立 NPU：

| 子系统 | 核心 | 用途 |
|--------|------|------|
| **应用处理器** | 双核 Cortex-M55 + 双核 BECO NPU | 主应用逻辑、AI 推理 |
| **主机处理器** | 双核 STAR-MC1 + 双核 BECO NPU | 传感器管理、低功耗常驻任务 |
| **协处理器** | BES 专有信号处理器 | 传感器融合、NN 辅助运算 |
| **DSP（可选）** | Tensilica HiFi 4 | 音频编解码、语音处理 |

> **双 NPU 组**意味着应用侧和常驻侧各有独立的 AI 推理能力，可并行运行不同模型——例如同时做血糖光谱推理 + 心率监测。

### 1.3 存储配置

| 类型 | 容量 |
|------|------|
| 片上 SRAM | **8.3 MB**（共享） |
| SIP Flash | 内置 |
| SIP PSRAM | 内置 |
| Boot ROM | 内置 |

8.3 MB SRAM 在 MCU 级芯片中属**顶级配置**（STM32N675 为 4.2 MB），可支撑中型 AI 模型的片上推理，无需外挂 RAM。

### 1.4 多媒体与显示

| 模块 | 规格 |
|------|------|
| GPU | 2.5D 矢量 GPU |
| LCD 控制器 | 3 层 alpha 混合 |
| MIPI DSI | 4-lane 或双 2-lane，最高 720p @ 24bpp |
| 图像解码 | 硬件加速 PNG 解码 |

## 2. 摄像头接口能力

### 2.1 CSI-2 定论：BES2800BP 不支持

**已确认：BES2800BP 不支持 MIPI CSI-2。**

官方 [Brief Datasheet](https://www.bestechnic.com/Uploads/keditor/file/20241010/20241010192510_70770.pdf) 的外设列表也印证了这一点——仅列出了 MIPI **DSI**（显示）和 SCI（串行通信），无 CSI / DVP / 任何摄像头接口。

此前投资者交流会上关于"BES2800 可带摄像头接口"的说法，实际情况是：AI 眼镜产品通过**外挂 ISP 芯片**桥接摄像头，ISP 与 BES2800BP 之间走 SPI / QSPI / USB 等通用接口，并非直连。

### 2.2 睿芯微百万像素 CIS 连接可行性

**睿芯微 CIS 实际参数**：

| 参数 | 规格 |
|------|------|
| 像素数 | **百万像素级** |
| 输出接口 | **MIPI CSI-2** + **DVP**（双接口支持） |
| 应用 | 高光谱成像 + 无创血糖监测 |
| 全速带宽 | 1MP × 16bit × 30fps ≈ **480 Mbps** |

**连接方案判定**：

| 方案 | 接口 | 带宽匹配 | 结论 |
|------|------|----------|------|
| MIPI CSI-2 直连 | ❌ BES2800BP 不支持 | - | **不可行** |
| DVP 直连 | ❌ BES2800BP 无并行摄像头口 | - | **不可行** |
| QSPI 桥接 | QSPI ~320 Mbps | ❌ 不足 480 Mbps | **不可行** |
| 外挂 ISP/FPGA | ISP→USB/SPI | ⚠️ 取决于桥接芯片 | **可行但成本/功耗不可接受** |

> **结论**：BES2800BP **无法直连睿芯微百万像素 CIS**，不适合作为无创血糖戒指/手表的主 SoC。如需使用 BES2800BP 的无线能力（BT 5.4 + Wi-Fi 6），只能作为**通信协处理器**，主控仍需带 CSI-2 的 SoC。

### 2.3 AI 眼镜量产案例印证

| 产品 | 摄像头连接方式 | 说明 |
|------|----------------|------|
| 理想 Livis | Camera → 研极微 ISP → BES2800 | BES2800 不直连摄像头 |
| Looktech | Camera → 星宸 SSC309QL ISP → BES2800 | ISP 内含 1.5T NPU |
| 传音 Infinix | Camera → 研极微 ISP → BES2800 | 成本最优 |

> 全部通过外挂 ISP 桥接，印证 BES2800BP 本身不具备摄像头接口。

## 3. AI / NPU 算力分析

### 3.1 BECO NPU 架构

| 项目 | 规格 |
|------|------|
| NPU 类型 | BECO（恒玄自研神经网络处理器） |
| 配置 | 双核 BECO NPU × 2 组（共 4 核） |
| 精确 TOPS | ⚠️ **未公开**（需 NDA） |
| 估算算力 | 参考同类 M55+NPU 方案，约 **0.3–0.6 TOPS** |
| 支持算子 | CNN 推理、向量运算 |
| 典型应用 | PPG 信号处理、心率/血氧、手势识别、语音唤醒 |

### 3.2 算力参考坐标

| 芯片 | NPU 类型 | 公开算力 | 架构级别 |
|------|----------|----------|----------|
| BES2800BP | BECO 双核×2 | 未公开（估 ~0.3–0.6 TOPS） | MCU 级 |
| STM32N675 | ST Neural-ART | **0.6 TOPS** (600 GOPS) | MCU 级 |
| Snapdragon W5+ | Hexagon 690 | **~1.1 TOPS** | AP 级 |
| 高通 AR1 Gen1 | Hexagon NPU | **~2 TOPS** | AI 眼镜专用 |

### 3.3 AI 推理能力评估

| 任务 | 可行性 | 说明 |
|------|--------|------|
| PPG 多波长光谱分析 | ✅ | NPU + HiFi 4 DSP 协同处理 |
| 轻量 CNN 推理 | ✅ | BECO NPU 原生支持 |
| MoE Transformer（小型） | ⚠️ | SRAM 足够，但算力可能受限 |
| 人脸/手势识别 | ✅ | AI 眼镜已量产验证 |
| 高光谱血糖模型 | ⚠️ | 需实测模型复杂度与推理延迟 |
| 语音唤醒/识别 | ✅ | HiFi 4 DSP + NPU 标配场景 |

## 5. 无创血糖场景选型建议

### 5.1 戒指形态 → STM32N675 首选（BES2800BP 已排除）

**BES2800BP 不适用原因**：不支持 CSI-2 / DVP，无法直连睿芯微百万像素 CIS。

**核心优势**：
- 6nm + MCU 架构 → 功耗极低，15mAh 电池可持续运行
- 8.3 MB SRAM → 可承载轻量级光谱 AI 模型
- BT 5.4 + Wi-Fi 6 → 数据同步无需额外无线芯片
- $5–10 → BOM 成本远低于竞品

**关键风险**：
- **⚠️ 致命风险**：睿芯微 CIS 是百万像素级，输出接口为 CSI-2 / DVP，全速采集带宽 ~480 Mbps。BES2800BP 官方文档未列出 CSI-2 或 DVP，SPI/QSPI 带宽不足。**如 NDA 确认无 CSI-2，BES2800BP 不适用于此场景**
- BECO NPU 精确算力未公开，血糖模型推理延迟需实测
- AI 眼镜量产案均通过外挂 ISP 桥接，非直连摄像头
- 开发生态不如 ST 成熟，SDK/工具链需评估

### 5.2 手表形态

| 定位 | 推荐 SoC | 理由 |
|------|----------|------|
| 专业血糖手表 | STM32N675 | 内置 ISP + 0.6 TOPS + MCU 级功耗 |
| 高端智能手表 | Snapdragon W5+ | 算力充足 + 生态完善 + Android 生态 |
| 性价比手表 | BES2800BP | 成本低 + 无线集成 + 功耗优 |
