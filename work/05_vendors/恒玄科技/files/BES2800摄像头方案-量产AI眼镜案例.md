# BES2800摄像头方案与量产AI眼镜案例

> **⚠️ 重要发现（2026-04-28）**：BES2800BP官方datasheet（Rev 0.41）中**未提及CSI接口**，仅支持MIPI DSI显示接口。此页面的量产案例可能使用的是BES2800系列其他型号，或存在未公开的CSI接口支持。**需要进一步确认量产产品使用的具体芯片型号和接口连接方式。**
>
> **核心发现**：BES2800**不内置Camera ISP**，但通过**外挂ISP芯片**支持摄像头功能，已有多个量产AI眼镜方案验证。
> **关键证据**：理想、传音等品牌的AI眼镜已量产，使用BES2800 + 外挂ISP方案。

## 一、BES2800的摄像头支持方式

### 1.1 核心结论

**BES2800本身不集成Camera ISP**，但具备以下能力：
- ✅ **MIPI CSI-2接口**（高速接口能力）
- ✅ 适配市场上主流的ISP芯片
- ✅ 形成带视频、拍照、图像识别的多模态智能眼镜系统

### 1.3 BES2800核心能力

**制程工艺**：

## 二、外挂ISP芯片方案

### 2.1 三大主流ISP供应商详细对比

#### 研极微（杭州）- SA62105X

**公司背景**：
- 总部位于杭州的低功耗智能影像芯片公司
- 旗下消费电子品牌"神眸"
- 安防摄像头IPC领域积累丰富经验
- ISP芯片原本设计用于安防监控，现被多家AI眼镜品牌采用

**代表芯片：SA62105X**

| 参数 | 规格 |
|------|------|
| 工艺制程 | 未公开（推测为成熟制程）|
| 图像处理 | 支持千万级像素摄像头 |
| 典型应用 | 理想Livis AI眼镜、传音Infinix AI Glasses Pro、L'Atitude 52°N等 |
| BOM成本 | **约70元**（理想Livis方案） |

**核心特点**：
- ✅ **成本优势明显**：与BES2800搭配，整体BOM成本可控制在千元以内，零售价能做到1500元以内，仅为高通AR1方案成本的几分之一
- ✅ **低功耗设计**：与BES2800的低功耗特性配合，可实现超长续航（理想Livis续航达18.8小时）
- ⚠️ **算法调校依赖**：图像算法由黑芝麻智能提供软件算法调教，成像效果高度依赖厂商的ISP调校能力

**已量产产品**：
- 理想Livis AI眼镜（恒玄BES2800 + 研极微SA62105X + Sony IMX681 1200万像素）
- 传音Infinix AI Glasses Pro
- L'Atitude 52°N
- 字节跳动在研AI眼镜（考虑中）

#### 星宸科技（SigmaStar）- SSC309QL

**公司背景**：
- 联发科（MTK）关联企业
- 安防IPC芯片领域深厚积累
- **2025年初发布专为AI眼镜打造的ISP芯片SSC309QL**

**代表芯片：SSC309QL**

| 参数 | 规格 |
|------|------|
| 工艺制程 | 先进制程（具体未公开）|
| 图像处理引擎 | 第四代自研ISP 4.0 |
| 拍照能力 | **12MP**（1200万像素）|
| 视频编码 | **4K@30fps** H.264/H.265，JPEG 12MP |
| 算力 | **1.5 TOPS NPU**（可跑本地分类/识别模型）|
| 封装技术 | **Chiplet封装，内置LPDDR4x** |
| 芯片尺寸 | 比高通AR1面积减少24%，宽度减少20% |
| 功耗 | 2M录像仅300mW，整机功耗600mW（较AR1下降50%）|
| AI识别功耗 | **仅30mW**（低功耗事件记录）|

**核心特点**：
- ✅ **专为AI眼镜设计**：非安防芯片改型，而是原生面向智能眼镜场景优化
- ✅ **高集成度**：采用Chiplet技术内置LPDDR4x，无需外挂DDR，显著缩小面积和成本
- ✅ **低功耗全天候录像**：支持全天候录像 + AI识别，功耗控制行业领先
- ✅ **NPU集成**：1.5T算力可支持本地低功耗智能应用（分类、识别模型）
- ✅ **先进影像技术**：支持3A（自动曝光/对焦/白平衡）、HDR、WDR、3DNR、LDC等

**已量产/对接产品**：
- Looktech AI眼镜（首发搭载）
- 正与多家品牌客户对接测试中

#### 富瀚微（Fullhan）- MC6350

**公司背景**：
- 国内领先的AI视觉芯片企业
- 深耕安防监控、车载摄像头ISP领域多年
- **2025年CES期间正式发布智能眼镜专用芯片MC6350**

**代表芯片：MC6350**

| 参数 | 规格 |
|------|------|
| 工艺制程 | **12nm** |
| 图像处理 | **12MP**（1200万像素），高性能ISP |
| 封装尺寸 | **8×8mm超小封装** |
| 内存 | **集成256MB DDR** |
| 功耗 | 典型视频拍摄功耗仅为市场主力方案的1/4 |

**核心特点**：
- ✅ **AI-ISP暗光增强**：基于多年ISP技术积累，AI ISP在暗光环境下表现优异，解决主流智能眼镜暗光拍摄不佳的痛点
- ✅ **超低功耗**：12nm工艺 + 低功耗架构，典型场景功耗仅为竞品1/4
- ✅ **超小尺寸**：8×8mm封装完美适配眼镜轻薄化需求
- ✅ **成熟度高**：MC635X系列已在智能锁、低功耗相机等领域大规模量产，芯片成熟度经市场验证
- ✅ **性价比突出**：对位消费级市场需求，填补"高通AR1太贵、低分辨率ISP效果差"的市场空白

**已量产/对接产品**：

- 眸刻MocoViso Lumina L1（富瀚微ISP方案，2K影像，推测QM102V）
- 正与国内外主要智能眼镜制造商接洽合作

## 四、与集成ISP的SoC方案对比

### 4.1 BES2800 + 外挂ISP vs 高通AR1

| 特性 | BES2800 + 外挂ISP | 高通AR1（集成ISP） | 对比结果 |
|------|-------------------|-------------------|----------|
| **成本** | 更低（约为AR1的几分之一）| 较高 | ✅ BES2800优势 |
| **功耗** | 更低，续航更长 | 相对较高 | ✅ BES2800优势 |
| **成像质量** | 经调校可接近AR1水平 | 原生支持，效果稳定 | ⚠️ AR1优势 |
| **集成度** | 需外挂ISP，调试难度大 | 单芯片，集成度高 | ✅ AR1优势 |
| **灵活性** | 可选择不同ISP芯片 | 固定ISP | ✅ BES2800优势 |
| **开发难度** | 较高（需调试ISP） | 较低（单芯片） | ⚠️ AR1优势 |
| **适用场景** | 对成本和续航敏感的AI眼镜 | 追求极致拍摄体验的产品 | - |

### 4.2 方案选择建议

**选择BES2800 + 外挂ISP**，如果：
- ✅ 成本敏感（消费级产品）
- ✅ 续航要求高
- ✅ 需要灵活选择ISP
- ✅ 有调试能力

**选择高通AR1**，如果：
- ✅ 追求极致成像质量
- ✅ 追求单芯片集成度
- ✅ 开发周期紧张
- ✅ 成本不敏感

### 6.3 睿芯微CIS + BES2800方案

