## 供应商档案信息架构

每个供应商的 `.md` 文件应包含以下 **5个模块**。前3个模块是供应商的**独立知识**，后2个模块是**项目关系**。

```
供应商档案
├── ① 公司档案         ← 谁是这家公司（独立于我们的客观信息）
├── ② 产品与技术体系   ← 他们做什么、技术路线是什么
├── ③ 市场与生态       ← 行业地位、客户、竞争对手
├── ④ 与我们的关系     ← 评估状态、关联度、关键结论
└── ⑤ 合作评估         ← 商务信息、联系人、交流记录（合作中才需要）
```

### ① 公司档案（必填）

> 目标：不看任何外部资料就能了解这家公司的基本面。

| 字段 | 说明 | 必要性 |
|------|------|--------|
| 一句话定位 | 用一句话说清做什么、核心能力 | 必填 |
| 公司全称 / 英文名 | — | 必填 |
| 成立时间 | — | 必填 |
| 总部 | — | 必填 |
| 员工规模 | 约 ×× 人 | 选填 |
| 上市/融资 | 上市代码 或 融资轮次/金额 | 选填 |
| 创始人/核心团队 | 关键人物背景 | 选填 |
| 官网 | URL | 必填 |
| 投资方 | 谁投了他们（链接到 [[07_investors/]]） | 选填 |

### ② 产品与技术体系（必填）

> 目标：理解这家供应商的技术路线图和产品能力边界，不只是列参数。

**包含：**

| 内容 | 说明 | 必要性 |
|------|------|--------|
| 产品矩阵 | 核心产品列表（型号、类别、关键参数、状态） | 必填 |
| 技术路线 | 他们的技术路线是什么？和同类相比有何不同 | 必填 |
| 技术原理 | 核心技术的原理描述（如有价值展开） | 选填 |
| 关键性能指标 | 该领域最重要的3-5个指标 | 选填 |
| 产品路线图 | 未来规划（如已知） | 选填 |

**产品矩阵格式：**

```markdown
| 产品/型号 | 类别 | 关键参数 | 状态 |
|-----------|------|----------|------|
| XXX | SoC | 双核240MHz, BLE5.0, 15mAh功耗 | 量产 |
```

### ③ 市场与生态（建议填）

> 目标：理解这家公司在行业中的位置，判断供应链稳定性和生态成熟度。

| 内容 | 说明 | 必要性 |
|------|------|--------|
| 行业地位 | 市场份额、排名、独特优势 | 建议填 |
| 主要客户 | 谁在用他们的产品 | 建议填 |
| 竞品供应商 | 同赛道还有谁 | 建议填 |
| 生态/社区 | 开发者生态、SDK、参考设计丰富度 | 选填 |
| 风险因素 | 供应链风险、地缘政治、技术路线风险 | 选填 |

### ④ 与我们的关系（必填）

> 目标：快速判断这家供应商对我们项目的价值。

| 字段 | 说明 |
|------|------|
| 关系类型 | 潜在主控 / 传感器供应商 / 技术参考 / 生态伙伴 / 竞品关联 |
| 关联度 | 高/中/低 + 一句话理由 |
| 关注重点 | 我们为什么要关注他们 |
| 潜在用途 | 在我们产品中的可能应用 |

### ⑤ 合作评估（合作中才需要）

> 目标：支撑采购和技术选型决策。

| 字段 | 说明 |
|------|------|
| 评估状态 | 评估中 / 样品测试 / 小批量 / 量产 |
| 评估结论 | 值不值得用？为什么？ |
| 商务条件 | 报价、交期、MOQ、付款 |
| 关键联系人 | 姓名/职位/联系方式 |
| 交流记录 | 日期 + 内容摘要 |

## 使用指南

### 新增供应商
1. 在 `05_vendors/` 下创建 `<供应商名>/` 目录 + `files/` 子目录
2. 创建 `<供应商名>.md`，按上方 5 模块架构填写
3. 至少完成 **①②④** 三个必填模块
4. 更新本 README 的分类索引和完整性评估

### 补充现有供应商
1. 对照上方完整性评估表，找到标记为 ⚠️ 或 — 的模块
2. 按优先级补充：②产品技术 > ③市场生态 > ⑤合作评估
3. 补充后更新完整性评估表

### files/ 目录结构

```
files/
├── 1_公司/         公司简介、年报、团队信息
├── 2_产品/         规格书、参考设计、手册、路线图
├── 3_评估/         Demo、测试报告、技术分析
├── 4_商务/         报价、合同、通讯记录
├── 5_拜访/         现场拜访记录
├── 6_新闻/         供应商新闻动态
└── 0_归档/         过期资料
```
