## 🚀 核心规格

### 处理器

**核心**: Arm® Cortex®-M55
- **主频**: 800 MHz
- **架构**: ARMv8-M + Helium DSP扩展
- **特色**: 首款引入 Arm Helium 向量处理技术的MCU

**性能**:
- **CoreMark**: 3616 CoreMark®，CoreMark® score 4.52
- **DSP能力**: 增强的DSP处理能力（Arm Helium / Arm MVE）
- **实时控制**: 低延迟中断响应
- **能效**: 高性能与低功耗平衡
- **缓存**: 32-Kbyte ICACHE，32-Kbyte DCACHE
- **安全**: TrustZone® MPU，NVIC
- **浮点**: 支持半精度/单精度/双精度FPU（向量和标量）

### 内存配置

**SRAM**:
- **总容量**: 4.2 Mbyte 连续SRAM
- **TCM**: 128-Kbyte 紧密耦合内存（带ECC，用于关键实时数据）
- **指令TCM**: 64-Kbyte 指令TCM（带ECC，用于关键实时例程）
- **备份SRAM**: 8-Kbyte 备份SRAM（在VBAT模式下活跃）
- **ECC**: 关键实时数据支持ECC纠错

**外部存储器接口**:
- **灵活外部存储器控制器**: 支持高达32位数据总线
  - SRAM, PSRAM, SDRAM/LPSDR SDRAM, NOR/NAND存储器
- **XSPI**: 支持串行PSRAM, NAND, NOR, HyperRAM™/HyperFlash™
  - 2个端口：XSPIM 8位和16位配置，最高200 MHz
- **加密引擎**: 集成加密引擎

**Flash**:
- **容量**: 待确认（具体容量见ordering information表）
- **读保护**: 支持
- **写保护**: 支持

### AI加速器

**ST Neural-ART Accelerator** (STM32N6x7系列专用):
- **性能**: 高达 600 GOPS（Giga Operations Per Second），288 MAC/cycle
- **频率**: 最高 1 GHz
- **专用硬件**: 用于DNN（深度神经网络）推理函数
- **流处理引擎**: 灵活的专用流处理引擎
- **实时加密/解密**: 支持实时加密/解密
- **权重解压缩**: 支持在线权重解压缩
- **适用**: 睿芯微血糖监测的AI模型推理

**应用场景**:
- ✅ 实时信号处理
- ✅ AI模型推理（如血糖预测）
- ✅ 复杂算法加速

### GPU和图形

**NeoChrom 2.5D GPU**:
- **类型**: 2.5D GPU
- **功能**: 缩放、旋转、Alpha混合、纹理映射、透视变换
- **Chrom-ART Accelerator**: DMA2D硬件加速器
- **Chrom-GRC**: GFXMMU图形内存管理单元
- **图形定时器**: GFXTIM
- **LCD-TFT控制器**: 最高XGA分辨率

**视频处理**:

**编码器**:
- **H.264编码器**: 硬件编码加速
  - 支持baseline profile、main profile、high profile（level 1到level 5.2）
  - 分辨率：1080p15和720p30
- **JPEG编解码**: 硬件编解码，支持MJPEG

**相机接口**:
- **并行和2通道CSI-2相机接口**
- **ISP（图像信号处理器）**: 三路并行管道处理同一输入流
  - 坏点校正、抽取、黑电平、曝光
  - 去马赛克、列转换、对比度
  - 裁剪、降采样、ROI
  - Gamma校正、YUV转换、像素打包

**应用**:
- 视频显示
- 图像处理
- UI渲染

### 安全特性

**硬件安全**:
- **Arm® TrustZone®**: 安全I/O、内存和外设
- **SESIP Level 3**: 物联网平台安全评估标准认证
- **Arm® PSA**: 平台安全架构认证
- **灵活生命周期方案**: 支持RDP（读保护）和密码保护调试
- **安全密钥配置**: 在OTP（一次性可编程）熔丝中安全配置客户密钥
- **安全启动**: ROM中的安全启动代码，解密和验证客户uROT（可更新根信任）
- **安全数据存储**: 硬件唯一密钥（HUK）
- **加密加速器**: 硬件支持的加密/解密引擎

## 🔌 外设和接口

### 通信接口

**USB接口**:
- **2个USB 2.0高速/全速设备/主机OTG控制器**
  - 其中一个支持UCPD USB Type-C® Power Delivery

**以太网**:
- **10-Mbit, 100-Mbit, 和 1-Gbit以太网**
  - 支持TSN（时间敏感网络）

**I2C/I3C**:
- **4个I2C Fm+接口**（SMBus/PMBus®）
- **2个I3C接口**

**SPI**:
- **6个SPI接口**（其中4个支持I2S）

**音频接口**:
- **2个SAI**（串行音频接口）
  - 支持4个DMIC（数字麦克风）

**UART/USART**:
- **5个USART**
- **5个UART**（ISO7816接口、LIN、IrDA，最高12.5 Mbit/s）
- **1个LPUART**（低功耗UART）

**SD/MMC**:
- **2个SDMMC接口**
  - MMC版本4.0
  - CE-ATA版本1.0
  - SD版本1.0.1

**CAN**:
- **3个FDCAN**（带TTCAN能力）

### 模拟外设

- **温度传感器**: 1个
- **ADC**: 2个12位ADC（最高5 Msps，最多20通道）
- **ADF滤波器**: 1个（带SAD）
- **MDF**: 1个（6个滤波器）

### 定时器和看门狗

- **32位定时器**: 4个（最多4个IC/OC/PWM或脉冲计数器，正交编码器输入，最高240 MHz）
- **16位高级电机控制定时器**: 2个（最高240 MHz）
- **16位通用定时器**: 13个（最高240 MHz）
- **16位低功耗定时器**: 5个（最高240 MHz）
- **看门狗**: 2个（独立看门狗和窗口看门狗）
- **SysTick定时器**: 1个

### DMA控制器

- **GPDMA**: 通用直接内存访问控制器
- **HPDMA**: 高性能直接内存访问控制器

## 📊 与其他MCU对比

### vs ESP32-P4

| 特性 | STM32N675 | ESP32-P4 |
|------|-----------|----------|
| **核心** | Cortex-M55 @ 800MHz | RISC-V @ 400MHz |
| **AI加速** | Neural-ART 600 GOPS | 待确认 |
| **SRAM** | 4.2MB | 无内置SRAM |
| **视频编码** | H.264硬件编码 | 无 |
| **成熟度** | 非常成熟 | 较新 |
| **成本** | 中等 | 低（$5-10） |
| **适用** | 高端演示 | 原型开发 |

### vs 高通Wear 4100+

| 特性 | STM32N675 | 高通Wear 4100+ |
|------|-----------|----------------|
| **核心** | Cortex-M55 | Cortex-A53 |
| **定位** | MCU（微控制器） | 应用处理器 |
| **AI加速** | Neural-ART | Hexagon DSP |
| **内存** | 4.2MB SRAM | 1GB LPDDR4 |
| **成本** | 低 | 高（$25-35芯片） |
| **适用** | 嵌入式/演示 | 智能手表 |
