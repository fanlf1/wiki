# JBD MicroLED + 光波导 AR 显示方案总结

## 方案定位

JBD 提供的是 AR/智能眼镜的**微型显示+光学投影完整方案**：MicroLED 微显示屏作为光源和图像源，配合衍射光波导镜片实现"透明镜片上显示画面"。

---

## 一、光学架构总览

```
                        AR 眼镜显示光路（侧视）

镜腿/铰链                     镜片（光波导）                  眼睛
┌──────────┐             ┌─────────────────────┐         ┌──────┐
│           │             │  出射光栅（EPE）     │         │      │
│  JBD      │  耦合光束   │    ↓  ↓  ↓  ↓  ↓   │────────>│  👁  │
│  MicroLED ├────────────>│  全反射光路          │  衍射   │      │
│  微显示屏  │  入射光栅   │  (光在玻璃内部弹射)  │  出射   │      │
│  0.13"    │             │                     │         │      │
│  640×480  │             │  波导基板 ~1mm       │         │      │
│  绿光530nm│             │  n ≈ 1.7-2.0        │         │      │
└──────────┘             └─────────────────────┘         └──────┘
   光源+图像源                  光传导+成像                    

三大组件：
① MicroLED 芯片 → 发光 + 生成像素图像
② 入射光栅     → 将光耦合进波导（改变角度满足全反射条件）
③ 出射光栅     → 将光衍射出波导（角度→空间位置映射 → 平面图像）
```

### 成像原理

光波导本质是**角度→空间位置的映射器**：

- MicroLED 芯片上每个像素发出的光，角度略有不同
- 不同角度的光在波导中弹射次数不同
- 出射光栅在不同位置将光衍射出来
- 每个像素对应一个独特的出射位置 → 组成平面图像

## 二、核心参数

### 2.1 MicroLED 微显示屏（当前量产主力）

| 参数 | JBD 单绿方案 | 备注 |
|------|-------------|------|
| 芯片尺寸 | 0.13 英寸 | 约3.3mm |
| 分辨率 | 640×480 (VGA) | 部分产品裁切为 640×200 |
| 发光波长 | ~530 nm（绿光） | InGaN 材料 |
| 峰值亮度 | 最高 800万 nits | 芯片级；经波导后眼镜端约 1000-1200 nits |
| 刷新率 | 60 Hz | |
| 光引擎体积 | 0.2 cc / 0.5 g | 业界最小级别 |
| 功耗 | 极低（单色微显示） | 相比 OLED/LCoS 有数量级优势 |

### 2.2 光波导参数（配合 JBD 方案的典型值）

| 参数 | 典型值 | 决定因素 |
|------|--------|----------|
| 视场角 (FOV) | 20°-27.5° | Even G1=20°, G2=27.5° |
| 透光率 | 98-99% | 光波导几乎不影响视物 |
| 光栅周期 | 270-400 nm | 由波长和折射率决定：Λ ≈ λ/n |
| 波导厚度 | ~1 mm | 工程参数（非光学共振） |
| 波导折射率 | 1.7-2.0 | 高折射率玻璃 |
| 成像距离 | ~2 m（虚拟画面） | 光学设计参数 |

### 2.3 光栅方程（核心设计公式）

```
Λ = λ / (n_eff × Δsinθ)

Λ     = 光栅周期（凹槽间距）
λ     = 光波长（绿光 ~530nm）
n_eff = 波导有效折射率（~1.7-2.0）
Δsinθ = 入射与衍射角度差的正弦值
```

**关键结论**：
- 光栅周期 Λ 与波长 λ 精确相关（Λ ≈ λ/n，约 300nm）
- 波导厚度 ~1mm 远大于波长（约2000倍λ），由工程折中决定
- 单色方案只需优化一组光栅周期；全彩需同时满足 R(~630nm)+G(~530nm)+B(~450nm) 三个波长

## 三、方案演进路线

### 3.1 当前：单色绿光（量产成熟）

```
┌──────────────┐     ┌──────────────┐     ┌──────────┐
│ JBD 0.13"    │     │ SRG衍射光波导 │     │ Even G1  │
│ 单绿MicroLED ├────>│ 单波长优化    ├────>│ 20° FOV  │
│ 640×480      │     │ 光栅~300nm    │     │ 1000nit  │
└──────────────┘     └──────────────┘     └──────────┘

优势：光栅设计简单、量产良率高、成本低
劣势：只有绿色文字/图标，无法显示图片/视频
```

### 3.2 下一代：全彩方案（2025-2026 推进中）

```
方案A：三片式（R+G+B 三个MicroLED → 合色 → 波导）
  → 已有光引擎产品"蜂鸟Ⅱ"

方案B：单片全彩（一个MicroLED芯片实现RGB）
  → "凤凰系列"在研，技术难度极高

方案C：单层全彩波导（一个光栅同时衍射RGB）
  → Dispelix Selva 2.0、歌尔光学在推进

核心挑战：光栅周期必须同时满足
  Λ(R=630nm) ≈ 370nm
  Λ(G=530nm) ≈ 310nm
  Λ(B=450nm) ≈ 260nm
  → 三者无法用同一周期完美匹配，需要特殊光栅设计
```

## 四、与竞品显示技术对比

| 维度 | **JBD MicroLED** | OLED 微显示 | LCoS | DLP |
|------|-------------------|------------|------|-----|
| 亮度 | **极强**（800万nit芯片级） | 弱（<3000nit） | 中 | 强 |
| 功耗 | **极低**（自发光，单色微瓦级） | 中（有机材料发光） | 高（需背光） | 高 |
| 体积 | **极小**（0.2cc） | 小 | 大 | 大 |
| 色彩 | 当前单绿，全彩在研 | 全彩成熟 | 全彩成熟 | 全彩成熟 |
| 对比度 | **极高**（自发光） | 高 | 中 | 中 |
| 响应速度 | **极快**（ns级） | 快（μs级） | 慢（ms级） | 快 |
| 配合波导 | **最佳适配**（高亮度补偿波导损耗） | 困难（亮度不够） | 可适配 | 可适配 |
| 成本 | 单色已量产可接受 | 成熟低成本 | 低 | 中 |
| 量产成熟度 | 单色量产，全彩推进中 | 成熟 | 成熟 | 成熟 |

**结论**：对于轻量级 AR 眼镜（文字/图标/通知显示），JBD 单绿 MicroLED + 光波导是当前**最优方案**——亮度足够穿透波导、功耗极低、体积最小。对于需要全彩的场景，等待 JBD 全彩方案成熟是合理路径。

JBD 提供 MicroLED 光引擎，但光波导需要另行选型：

## 六、集成设计要点

### 6.1 光机位置

```
眼镜俯视图：

              ┌──────────────────────────┐
              │        波导镜片           │
              │                          │
  ┌────────┐  │                          │  ┌────────┐
  │ 镜腿    │  │                          │  │ 镜腿    │
  │         │  │                          │  │         │
  │ 电池    │  │                          │  │ JBD光机 │← 此处
  │ 主板    │  │                          │  │ 0.2cc   │
  └────────┘  │                          │  └────────┘
              └──────────────────────────┘
```

- JBD 光引擎仅 0.2cc/0.5g，可放置于单侧镜腿铰链处
- 光束从镜腿射入波导镜片边缘（耦合光栅位置）

### 6.2 处方镜片集成

Even G1 的创新点：将**度数矫正直接集成到波导镜片**中（数字化刻蚀），而非叠加一层矫正镜片。这是 Karl Guttag 评价为"首次在 AR 眼镜上看到"的集成方式。

### 6.3 功耗预算

单绿 MicroLED 微显示功耗在微瓦-毫瓦级，对眼镜整体续航几乎不构成负担。主要功耗消耗在蓝牙通信和传感器端。
