 MARD 的全称
MARD = Mean Absolute Relative Difference
👉 平均绝对相对误差
这是血糖监测领域最重要、最常用的准确性指标之一，尤其用于：

CGM
无创血糖
可穿戴血糖设备
与“血浆血糖”或“指尖血糖”对标


✅ MARD 的直观含义（一句话）

MARD 表示：设备测得的血糖值，平均来看，与参考血糖值相差百分之多少。

数值越小，说明设备越准。

✅ 用一个非常具体的例子理解
假设某一时刻：

实验室血浆血糖 = 10.0 mmol/L
你的设备测得 = 9.3 mmol/L

相对误差：
∣9.3−10.0∣/10.0=7%|9.3 - 10.0| / 10.0 = 7\%∣9.3−10.0∣/10.0=7%
👉 如果很多点算下来平均是 7.33%
👉 MARD = 7.33%

二、结合你给的对比信息，逐条解释 MARD
1️⃣ 无创 / 可穿戴连续血糖（你第一组描述）

以血浆血糖为对标基准（MARD 7.33%）

✅ 这表示：

和“实验室级血浆血糖”相比
平均误差 约 7.33%
在无创检测里，这是一个非常不错、偏高端的水平

同时：

100% 数据位于 ISO 15197 共识误差网格 Zone A & B3

说明：

没有“危险性错误”
即使有误差，也不至于导致错误临床决策


2️⃣ 指尖采血（血糖仪）

MARD ≈ 5.05%

✅ 含义：

比多数无创方案更准
因为是侵入式直接血样测量

但代价是：

侵入性
一次性耗材
用户体验差

👉 这是目前很多无创方案想“追赶或接近”的标杆

3️⃣ 连续血糖监测系统（CGM，微创）

MARD 9.2%–16.6%

✅ 含义：

误差更大，原因包括：

组织液 vs 血浆的生理延迟
传感器老化


但胜在：

连续监测
成熟产品



👉 这也是为什么：

无创 + 低 MARD + 连续
是行业的“终极目标”

三、顺带回答你前一个问题：
👉 这种血糖系统是“如何训练出来的”？（概念级，不涉及商业机密）
✅ 1️⃣ 数据采集（最核心）
同时采集三类数据：


参考标签（Ground Truth）

实验室临床生化分析仪
血浆血糖（金标准）



输入信号

光学信号（PPG、多波长）
血管实时成像特征
时间、心率、温度等



同步时间戳

保证输入与血糖对齐




✅ 2️⃣ 构建训练样本（监督学习）
训练样本形式是：
输入 X：
  多通道光学/成像/时序特征（过去几分钟～几十分钟）

标签 y：
  同一时间点的血浆血糖值

👉 这一步决定了它是“监督学习”

✅ 3️⃣ 模型训练（如 Transformer）

模型结构：

Transformer / LSTM / 混合模型


目标：

最小化预测血糖值与真实血糖值的误差


损失函数：

MSE / MAE / 相对误差 等




✅ 4️⃣ 验证与评价
核心指标就是你看到的那些：

MARD
ISO 15197 共识误差网格（Zone A / B / C / D / E）
Bland–Altman 分析（论文里常见）


✅ 5️⃣ 上设备部署

模型压缩
边缘计算
实时推理
连续输出血糖曲线


✅ 最后一句“行业级理解”

MARD x.x% 表示：这个血糖系统“平均偏差有多大”，
是判断无创/CGM 能不能“上临床、谈价值”的第一指标。
