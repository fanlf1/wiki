## 📖 什么是LLM Wiki？

LLM Wiki是一种**持久化、复合式知识库**模式：

- **不同于传统RAG**：不是每次查询时重新检索和综合
- **知识编译一次**：摄取时提炼整合，查询时直接使用
- **持续演进**：每次新源都会更新相关页面，标注矛盾

### 核心架构

```
┌─────────────────────────────────────────────────┐
│  三层架构                                        │
├─────────────────────────────────────────────────┤
│  1. Raw Sources  (原始素材，不可变)             │
│  2. Wiki         (LLM生成的markdown，LLM维护)    │
│  3. Schema       (CLAUDE.md规则)                │
└─────────────────────────────────────────────────┘
```

## 🎯 用途

本wiki当前可用于：

- 📚 **研究笔记**：论文、文章的渐进式整合
- 💡 **概念探索**：深入理解某个领域
- 🔍 **源文件追踪**：追溯每个主张的来源
- 🧠 **知识连接**：发现概念间的关联

## 🚀 快速开始

### 摄取新源

1. 将文件放入 `raw/` 对应子目录
2. 告诉LLM："摄取 `raw/xxx/yyy.md`"
3. LLM会自动：
   - 阅读并讨论关键要点
   - 创建源文件摘要
   - 更新相关概念/实体页
   - 更新索引和日志

### 查询知识

直接提问，LLM会：
1. 搜索 `index.md` 找到相关页面
2. 深入阅读相关内容
3. 综合回答并引用来源
4. （可选）将答案存为新的查询页
