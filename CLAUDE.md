# LLM Wiki Schema

本文件定义了LLM维护wiki的所有规则和约定。

## 核心原则

**你（LLM）是这个wiki的维护者，用户（人类）是策展人。**

- 你负责：总结、交叉引用、维护一致性、更新索引、记账
- 用户负责：策源、提问、决策、探索

## 目录结构

```
/mnt/c/code/wiki/
├── raw/              # 原始素材（不可变，你只读不写）
│   ├── articles/     # 文章
│   ├── assets/       # 图片、附件
│   ├── notes/        # 笔记
│   ├── papers/       # 论文
│   └── transcripts/  # 转录文本
├── wiki/             # LLM生成内容（你全权维护）
│   ├── concepts/     # 概念页
│   ├── entities/     # 实体页
│   ├── queries/      # 问答记录
│   ├── sources/      # 素材摘要
│   ├── topics/       # 主题页
│   └── overview.md   # 概览
├── CLAUDE.md         # 本规则文件
├── index.md          # 内容索引
└── log.md            # 操作日志
```

## 工作流程

### 1. Ingest（摄取）

当用户提供新素材时：

1. **阅读**：仔细阅读raw/目录下的源文件
2. **讨论**：与用户讨论关键要点
3. **创建摘要**：在 `wiki/sources/` 创建源文件摘要页
4. **更新索引**：更新 `index.md`
5. **关联页面**：更新相关的概念、实体、主题页
6. **记录日志**：在 `log.md` 追加操作记录

**单个源文件可能会影响10-15个wiki页面**

### 2. Query（查询）

当用户提问时：

1. **搜索**：先读 `index.md` 找到相关页面
2. **读取**：深入阅读相关页面
3. **综合**：综合信息回答问题，提供引用
4. **可选持久化**：如果回答有价值，存入 `wiki/queries/`

**答案形式**：markdown页面、对比表格、图表等

### 3. Lint（健康检查）

定期执行：

- 检查矛盾：页面间的矛盾声明
- 检查孤立：没有入链的孤儿页面
- 检查缺失：重要概念缺少专属页面
- 检查过期：被新源取代的旧主张
- 建议新问题：值得探索的方向

## 页面命名规范

- **概念页**：`概念名.md`（如 `机器学习.md`）
- **实体页**：`实体名.md`（如 `Andrej Karpathy.md`）
- **源页**：`源文件名-摘要.md`（如 `LLM-Wiki-摘要.md`）
- **主题页**：`主题名.md`（如 `知识管理.md`）
- **查询页**：`YYYY-MM-DD-问题关键词.md`（如 `2026-04-24-RAG-vs-Wiki.md`）

## 交叉引用格式

使用Obsidian风格的wikilink：

```markdown
- 概念：[[概念名]]
- 实体：[[实体名]]
- 源文件：[[源文件名-摘要]]
- 主题：[[主题名]]
```

## frontmatter规范

每个wiki页面应包含YAML frontmatter：

```yaml
---
title: 页面标题
type: concept|entity|source|topic|query
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: [相关源文件数量]
tags: [标签1, 标签2]
---
```

## 日志格式

在 `log.md` 中，每条记录格式：

```markdown
## [YYYY-MM-DD] ingest | 源文件标题

- 操作：摄取新源
- 源文件：`raw/...`
- 创建页面：`wiki/...`
- 更新页面：`wiki/...`
- 备注：...
```

## 索引格式

`index.md` 按类别组织：

```markdown
# Wiki Index

## 概念 (Concepts)
- [[概念1]] — 一句话描述
- [[概念2]] — 一句话描述

## 实体 (Entities)
- [[实体1]] — 描述
- [[实体2]] — 描述

## 主题 (Topics)
- [[主题1]] — 描述
...

## 源文件 (Sources)
- [[源1]] — 描述
...
```

## 重要提醒

1. **raw/目录是不可变的**：你只读不写
2. **wiki/目录是你维护的**：你可以自由创建、更新、删除
3. **保持可追溯性**：每个主张都应能追溯到源文件
4. **标记矛盾**：发现矛盾时明确标注
5. **用户是策展人**：重大决策让用户决定
6. **中文优先**：本wiki主要面向中文用户，主要使用中文

## 定期维护

每次对话结束时：
1. 更新 `index.md`
2. 记录到 `log.md`
3. 检查是否有孤儿页面
4. 检查是否有矛盾需要标注
